CX Matrix Decompositions for Tumour Classifications¶

In this example, we are going to replicate one of the experiements from the paper . We have a dataset of gene expressions for about 5,000 genes from 31 different tumours. Their are 3 different types of tumours, and the question we want to answer is: can we determine the type of tumour from just a handful of the 5,000 different genes?

We are going to do this by picking genes that have high leverage scores.

First step is to import the data:

Theres a function in SpaLor for calculating the leverage scores. It requires a rank, but this is not the same as the number of columns we hope to sample. The leverage scores are how important a given column is when we are constructing a rank r approximation of the matrix. We are going to calculate and plot them here:

In [16]:
from spalor.util.randomized_matrix_computations import leverage_scores_QR
from matplotlib import pyplot as plt
%matplotlib notebook

ls=leverage_scores_QR(data_matrix.transpose(),r=3)
plt.plot(ls,'o')
plt.show()

A good way to think about this plot is that the genes in this plot that have a much larger leverage score than average are the ones that contain the most information. When we fit our data to a CX model from SpaLoR, its going to randomly sample genes with a probability proportional to the leverage score squared.

And those three all look pretty distinct, so we should be able to cluster the genes using an unsupervised K-Means clustering algorithm. We're going to do this for our 25 genes we picked with CX, and all 5,520 genes. In both cases, they perfectly cluser the 31 tumours into their types, but the first one does it with significantly less data!
